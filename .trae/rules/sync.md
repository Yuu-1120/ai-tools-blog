# 文档同步规则

## 触发条件

当完成以下操作时，**主动询问**用户是否需要同步更新文档：

1. **新功能完成** → 询问更新 `.trae/plan/roadmap.md` + `.trae/plan/progress.md`
2. **踩坑并解决** → 询问更新 `.trae/plan/pitfalls.md`
3. **需求变更** → 询问更新 `.trae/plan/roadmap.md`

## 询问方式

完成上述操作后，直接提问：

```
「需要同步更新文档吗？」
```

- 用户回答 **"是" / "需要" / "更新"** → 立即更新对应文档
- 用户回答 **"不" / "否" / "暂时不用"** → 不更新
- 用户**未回复** → 不更新

## 更新原则

- 只更新**确实完成**的内容，未完成的不要提前标记
- 简洁记录，无需过度展开
- 优先更新 `.trae/plan/progress.md`（当前状态），`.trae/plan/roadmap.md` 按需调整
